# Deep Client Analysis Pipeline

Полный цикл анализа клиента для подготовки к продаже. Запускается по ID сделки или контакта.

## Как запустить

```
/assistant
```
Затем указать ID сделки, например: «Проанализируй сделку ID 12345»

---

## Фаза 1 — Инициализация

1. Получи ID сделки/контакта от менеджера
2. Определи тип запроса (сделка или контакт)
3. Загрузи agent-конфигурацию из `.agents/agents/coordinator.md`

---

## Фаза 2 — Сбор данных (DataCollector)

Агент: `.agents/agents/data-collector.md`
Skill: `bitrix24-deep-crm-analyzer`

// turbo
4. Вызови Bitrix24 MCP `b24-dev-mcp` — получи карточку сделки
5. Из карточки извлеки ID компании и ID контакта
// turbo
6. Параллельно вызови:
   - Карточку компании (по company_id)
   - Карточку контакта (по contact_id)
   - Timeline сделки
   - Активности (звонки, письма, встречи)
// turbo
7. Получи транскрипты звонков CoPilot
8. Получи историю переписок (Open Lines, почта, чаты)

---

## Фаза 3 — Исследование сайта (Researcher)

Агент: `.agents/agents/researcher.md`
Skill: `client-website-researcher`

**Запускается параллельно с Фазой 2, если URL компании уже известен.**

// turbo
9. Извлеки URL сайта из карточки компании
10. Открой сайт через Browser Tool
11. Проанализируй: главная, о компании, продукты/услуги, новости, контакты
12. Сформируй структурированное резюме сайта

---

## Фаза 4 — Анализ (Analyst)

Агент: `.agents/agents/analyst.md`
Skills: `call-transcript-analyzer` + `pain-point-extractor`

**Старт после завершения Фаз 2 и 3.**

13. Получи все данные от DataCollector и Researcher
14. Проанализируй транскрипты — извлеки боли, пожелания, возражения с цитатами
15. Определи стиль коммуникации и принятия решений
16. Приоритизируй инсайты по влиянию на сделку

---

## Фаза 5 — Стратегия (Strategist)

Агент: `.agents/agents/strategist.md`
Skill: `sales-strategist`

**Финальная фаза. Старт после завершения Фазы 4.**

17. Получи все инсайты от Analyst
18. Сформируй финальное резюме в строгом Markdown:
    - AI-Обзор клиента
    - Ключевые боли и пожелания (с цитатами)
    - Сильные стороны и возможности
    - Авторитетные рекомендации менеджеру
    - План разговора (открывашка → вопросы → аргументы → возражения → close)
19. Сохрани результат в виде Markdown файла в `Z:\!_STUFF\ALTAIR\assistant-project\ready_leads\[ID сделки]_[Название сделки].md`
20. Выведи результат менеджеру

---

## Фаза 6 — Генерация лендинга (Designer)

Скилл: `skills/report-landing-generator/skill.md` и правила из `design/ALTAIR-tokenfile.md`

**Старт после завершения Фазы 5 и сохранения Markdown-отчета.**

21. Опираясь на только что созданный Markdown-отчет, сгенерируй HTML-лендинг строго по обновленным правилам (тёмный матовый контейнер, 3 гигантские переливающиеся фигуры на фоне, адаптация текста).
22. Создай новую папку по номеру лида в `Z:\!_STUFF\ALTAIR\assistant-project\design\[ID сделки]\`.
23. Сложи сгенерированный `index.html` и `style.css` в эту папку.
24. Уведоми менеджера об успешной генерации лендинга со ссылками на готовые файлы.
